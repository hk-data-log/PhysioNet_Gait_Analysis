## 📌 프로젝트 상태: 분석 및 최적화 진행 중 (In Progress)
본 프로젝트는 **PhysioNet Gait in Elderly Adults 데이터셋**을 정복하기 위해 매일 머리를 싸매고 있는 기록입니다. 
단순히 예쁜 결과만 올리는 곳이 아니라, 시행착오를 거쳐 정답을 찾아가는 **리얼한 분석 과정**을 담고 있습니다.

## 📂 저장소 구조
- **`archive_trash/`**: 3일간의 사투가 담긴 시행착오 기록 (에러 뭉텅이와 변수명 혼선의 흔적들)
- **그 외 파일들**: 현재 분석 고도화 및 파이프라인 최적화가 진행 중인 코드셋

## 💡 삽질 기록: 수천 줄의 코드를 갈아엎으며 배운 것들
매일 에러와 싸우며 뼈저리게 느끼고 있는 실전 원칙들입니다.

1. **"에러는 발견 즉시 검거한다"**
   - 에러 난 셀을 해결하지 않고 넘기면, 나중에 에러가 뭉텅이로 돌아와 손을 쓸 수 없게 됨을 배웠습니다. '나중에'는 없습니다. 지금 해결하는 것이 가장 빠릅니다.

2. **"변수명은 데이터의 신분증이다"**
   - 변수명을 대충 지으면 분석 중에 내가 만든 변수도 못 찾는 미궁에 빠지게 됩니다. 명확한 이름이 곧 분석의 속도이자 실력임을 깨달았습니다.

3. **"나의 뇌보다 주석을 믿자" (어제의 나는 내가 아니다)**
   - 자고 일어나면 내 코드조차 남의 코드처럼 생소해집니다. 나의 기억력을 과신하기보다, 미래의 나를 위해 상세한 주석을 남기는 시스템이 필수임을 절감합니다.

4. **"오늘의 삽질은 내일의 자산이다"**
   - 지금 쌓여있는 `archive_trash`의 기록들은 단순한 실패가 아닙니다. 이 과정에서 얻은 데이터 핸들링 노하우는 향후 국건영 데이터 등 대규모 프로젝트를 정복할 가장 강력한 밑거름이 될 것입니다.
